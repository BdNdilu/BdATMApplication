# BdATMApplication 💳🏦

## 📌 Overview
ATM simulation inspired by my TUT lecturer’s banking app example, customized to apply everything I’ve learned in Java:
- Composition
- Inheritance
- Polymorphism
- Interfaces
- Abstract classes
- Exception handling
- ArrayList collections

This project demonstrates practical Object‑Oriented Programming concepts while simulating real banking operations.

---

## ⚙️ Features
- Create and manage multiple accounts
- Deposit, withdraw, and check balances
- Handle insufficient funds with custom exceptions
- Demonstrate OOP principles in Java
- Store accounts in an `ArrayList` for flexible management

## 💡 Concepts Demonstrated
- Composition → `Account` has an `AccountHolder`
- Inheritance → `CheckingAccount` and `SavingsAccount` extend `Account`
- Polymorphism → Accounts share common behaviors but override specifics
- Interfaces → `Transactional` implemented by `Account`
- Abstract classes → `Account` defines base structure
- Exception handling → `InsufficientFundsException` used in withdrawals
- Collections → `ATM` manages accounts with `ArrayList`
